# 거래명세서 정리 도구 — 사용 설명서

거래명세서를 직접 입력·관리하는 1인용 데스크톱 프로그램입니다.
**거래처·명세서·품목·카테고리를 자유롭게 등록/수정/삭제**하고, **결제일을 자동 계산**해 줍니다.
인터넷·로그인·서버가 필요 없으며 모든 데이터는 이 PC 안에만 저장됩니다.

---

## 설치 / 실행

1. 받은 `DueLedger-x.x.x-portable.exe` 파일을 원하는 폴더에 둡니다.
2. **더블클릭하면 바로 실행**됩니다. (설치 과정 없음)
3. 첫 실행 때 윈도우가 _"Windows의 PC를 보호했습니다"_ 또는 _"알 수 없는 게시자"_ 경고를 띄울 수 있습니다.
   → **추가 정보 → 실행** 을 한 번 눌러 주세요. (사내용이라 서명을 하지 않아 뜨는 정상 경고입니다.)
4. 업데이트는 **새 .exe 파일로 교체**만 하면 됩니다. 기존 데이터는 그대로 유지됩니다. (자동 업데이트 없음)

> .exe 파일을 다른 폴더로 옮기거나 새 버전으로 바꿔도 입력한 데이터는 사라지지 않습니다.

---

## 데이터 저장 위치 / 백업 (중요)

- 데이터는 한 개의 파일에 저장됩니다:
  **`%APPDATA%\DueLedger\dueledger.db`**
  (윈도우 탐색기 주소창에 `%APPDATA%\DueLedger` 를 입력하면 폴더가 열립니다.)
- **백업** = 그 폴더의 `dueledger.db` 파일을 복사해 두면 됩니다.
- 서버에 따로 저장되지 않으므로, 중요한 데이터는 **가끔 이 파일을 백업**해 두시길 권합니다.

## 화면 구성

상단에 5개 탭이 있습니다.

| 탭 | 하는 일 |
|---|---|
| **홈** | 다가오는 결제 한눈에 — 연체 / 임박(7일 이내) / 예정 / 총 미지급 금액, 챙길 결제 목록, 최근 입력·수정 내역 |
| **명세서** | 입력한 거래명세서 목록(표). 정렬·필터·검색, 새 명세서 입력/수정/삭제 |
| **달력** | 결제일을 월간 달력으로. 미지급이 있는 날은 빨갛게 표시 |
| **거래처** | 거래처 등록·수정·삭제, 결제조건 설정 |
| **카테고리** | 품목 분류(식자재·포장재 등) 관리 |

> 앱을 켜면 **홈**이 먼저 열려, "언제 누구한테 얼마 줘야 하는지"를 바로 보여줍니다.
> 결제상태는 색으로 구분됩니다 — 미지급(빨강) · 지급예정(노랑) · 지급완료(회색), 결제일 지난 미지급은 강조됩니다.

## 사용법

### 1단계. 거래처 등록 (먼저 하면 편합니다)

**거래처** 탭 → 이름 입력 → **결제조건** 선택 → **추가**

결제조건은 결제일 자동 계산에 쓰입니다.

| 결제조건 | 뜻 | 예시 |
|---|---|---|
| 없음 | 결제일 계산 안 함 | — |
| 발행일 +N일 (net) | 발행일로부터 N일 뒤 | 30 → 6/16 발행이면 결제일 **7/16** |
| 매월 N일 | 매달 N일이 결제일 | 25 → 결제일 매월 **25일** |

> "매월 N일"에서, 발행일이 이미 그 달 N일을 지났으면 **다음 달** N일이 됩니다.
> N일이 그 달에 없으면(예: 31일, 2월) 그 달 **마지막 날**로 맞춰집니다.

### 2단계. 명세서 입력

**명세서** 탭 → **+ 새 명세서** → 거래처·발행일·결제상태·비고 입력 → 품목 줄 입력 → **저장**

품목은 한 명세서에 **여러 줄** 넣을 수 있습니다. (**+ 품목 추가**)

- **공급가액이 기준 금액**입니다. 수량·단가는 선택 입력 — 총액만 알면 공급가액만 넣어도 됩니다.
  (수량이 "2박스"처럼 숫자가 아니면 비워 두세요.)
- **부가세·합계는 자동 계산**됩니다:
  - 과세: 부가세 = 공급가액 × 10% (반올림)
  - 면세: 부가세 = 0
  - 합계 = 공급가액 + 부가세
- 저장하면 목록에 나타나고, **결제일이 거래처 결제조건에 따라 자동으로 채워집니다.**
- **결제일을 직접 정하고 싶으면** "직접 지정"에 체크하고 날짜를 입력하세요. (체크 안 하면 자동 계산)
- 거래처/카테고리가 없으면 이 화면에서 **"+ 새 거래처" / "+ 새 카테고리"로 바로 추가**할 수 있습니다.

### 3단계. 조회 (표)

- **정렬**: 표 머리글(거래일자·거래처·카테고리·공급가액·합계·결제상태·결제일) 클릭. 다시 누르면 역순.
- **필터**: 거래처 · 결제상태 · 월(년-월) 선택.
- **검색**: 품목명·거래처명·비고에 들어간 단어로 검색.
- 각 줄의 **수정 / 삭제** 버튼으로 그 명세서를 고치거나 지울 수 있습니다.

### 카테고리 관리

**카테고리** 탭에서 추가·이름변경·삭제.

- 기본 제공: 식자재 / 포장재 / 소모품 / 위생용품 / 기타. 필요하면 자유롭게 추가하세요.
- **사용 중인 카테고리는 삭제되지 않습니다** — 몇 건이 쓰고 있는지 알려줍니다. 그 품목들의 분류를 바꾼 뒤 삭제하세요.
- 이름을 바꾸면 그 카테고리를 쓰는 품목 전부에 반영됩니다.

### 결제상태 vs 결제일

- **결제일** = "언제까지 내야 하나" (거래처 조건으로 자동 계산)
- **결제상태** = "냈나" (미지급 / 지급예정 / 지급완료를 직접 선택)
- 둘은 별개입니다.

---

## 자주 묻는 것

- **데이터가 어디 저장되나요?** 이 PC의 `%APPDATA%\DueLedger` 폴더에만. 인터넷·서버는 쓰지 않습니다.
- **다른 PC에서도 쓰려면?** 1대용 도구입니다. 옮기려면 위 `dueledger.db` 파일을 복사하세요.
- **금액 단위는?** 원(정수)으로 저장됩니다.
- **로그인이 없나요?** 네, 혼자 쓰는 도구라 계정·암호가 없습니다.
